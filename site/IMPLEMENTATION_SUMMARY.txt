STATIC PAGE GENERATION

  • dynamicParams = true: Remaining metros generated on-demand (ISR)
  • Build time optimization: ~10-15s vs building all 4,283 metros
  • Score-based threshold: Prioritizes important metros

RESPONSIVE DESIGN
  • Mobile-first approach
  • 2-4 column layouts depending on section and viewport

  • Touch-friendly interactive elements

DATA HANDLING
  • Graceful degradation for missing sections
  • Only renders sections with data (no empty cards)
  • Handles metros with minimal data (e.g., 0.3 score + population)
  • Safe optional chaining throughout

DATA REQUIREMENTS
================================================================================

From @/lib/data exports:
  • getAllMetros() - returns Metro[] for similarity lookup
  • getMetroDetail(slug) - returns MetroDetail | null
  • formatPop(n) - formats population (20.1M, 500K)
  • formatMarketCap(n) - formats currency ($2.0T, $500B)
  • regionColors - maps region names to CSS color variables

JSON Structure (public/data/details/{slug}.json):
  • metro object with 16 dimensions
  • Optional: teams, universities, culture, skyscrapers, luxury, events, marketCap
  • All sections work with partial data
